O HTTP é um protocolo web que permite comunicação entre pontas na internet. Ele possui duas versões principais, o HTTP 1.1 e 2.0.
Suponha uma solicitação bem sucedida, do tipo Request realizada pelo cliente utilizando o método GET. Qual o statuscode da mensagem response recebida pelo cliente? 200 OK
Quais as principais atualizações de versões quando se trata do HTTP 1.1 e 2.0, respectivamente? Introdução do CORS e Keep-alive no HTTP 1.1. Na versão 2.0 a transferência das requisições via streaming dentro de uma mesma conexão TCP e a feature PUSH lado server/cliente.
O HTTP possui a propriedade Stateless. Marque a alternativa correta: Um reenvio de uma requisição será respondida pelo servidor normalmente.
Por que proteger as mensagens HTTP com SSL? Evitar fishing e manter privacidade dos usuários através da criptografia.
Sabendo que uma conexão WebSocket utiliza toda a infraestrutura criada para o protocolo HTTP, indique qual afirmação relacionada ao seu funcionamento é uma afirmação CORRETA: Através da troca de mensagens HTTP que o cliente indica ao servidor a utilização do protocolo WebSocket. Por sua vez, o servidor realiza o upgrade da conexão utilizando o WebSocket API.
Em um cenário anterior ao protocolo Websocket, conexões HTTP eram utilizadas para suprir as demandas de aplicações baseadas em navegadores (browsers). Dentro deste contexto, indique os problemas existentes em uma conexão HTTP para cenário de comunicação constante com o servidor (caso das web browsers app). Alto overhead, conexões subjacentes e mapeamento das conexões por script pelo lado do cliente.
Há alguns recursos que permitem uma melhor experiência do usuário em termos de navegabilidade, os cookies e cache. Os servidores web cache também são conhecidos como proxy servers, o cache permite que conteúdos mais comumentes solicitados sejam armazenados no cache server. Dessa forma, além de outras vantagens melhorar a experiência do usuário, as principais vantagens da utilização de caches estão na redução do tempo de resposta e redução de tráfego originado pela comunicação HTTP, apesar dos cookies possuírem diversas vantagens, seus dados podem ser utilizados por um atacante malicioso em uma eventual invasão ao sistema.
Uma mensagem HTTP é formada pelo Request ou Status line, Header Line, assim como a Entity. Dentro do Entity são definidos o entity header e entity body. Com relação a estes dois campos é correto afirmar que: O entity header possui os metadados relacionados ao payload, contido no entity body, que está sendo enviado/recebido pelo servidor.
Quem garante a autenticidade das chaves de outras entidades? CA - Certification Authority ou Entidades certificadoras.
